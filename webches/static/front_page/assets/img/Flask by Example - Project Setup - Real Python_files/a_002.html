        This is cool.

        Virtualenvwrapper provides a "mkproject" command. It's the exact same as the mkvirtualenv command but it also creates a .project text file in the root of the virtualenv which contains the dirpath of the project source code.

        With this file in place, workon will automatically cd you to the directory.

        Another tip I like is when you check out an existing project from version control that you want to use in a virtualenv. Do

        mkvirtualenv -a $ PWD -r requirements.txt project-name

        This creates a virtualenv, sets the current dir as the project directory then installs requirements from the requirements.txt

        A useful change to this is when you package your flask apps with setuptools - replace the -r requi... With -i'-e' (note the lack of spaces, it's important!) - this will install the package in editable mode and any dependencies into the venv.

                      John Shelley Omer Khan • 2 years ago

                      One thing I've done is create a .env file that contains my env commands for a project. For instance:
                      ~~~~~~
                      #!/bin/bash

                      virtualenv ~/.virtualenvs/foo-flask-app

                      source ~/.virtualenvs/foo-flask-app/bin/activate

                      export APP_SETTINGS="config.DevelopmentConfig"

                      ~~~~~

                      Then from your project run:

                      $ source .env

                      It will execute the file. That way you don't have clashing APP_SETTING variables for different projects, which is the whole purpose of virtualenv. I've seen this done before from other developers but would appreciate what other people think about it as well.

                      (Disclaimer: I ran into issues with foreman from heroku because of this, and ended up having to rename it temporarily)

                      (You also can have an if statement that checks if your virtualenv is already a directory and if it isn't to create it else leave it alone)
